# v0.0.1 Release Notes

## Release Positioning

`v0.0.1` is a local alpha release of `zoo-keeper-server`.

This release is meant for:

- local development
- trusted single-user setups
- trusted LAN environments where the operator understands the security and
  runtime limits

This release is not positioned as a hardened multi-tenant or public-internet
deployment.

## Included In v0.0.1

- single-model HTTP server built on Drogon
- OpenAI-compatible subset for `POST /v1/chat/completions`
- SSE streaming
- optional in-memory sessions
- API key authentication
- basic `/metrics` and `/healthz` endpoints
- optional browser test UI when built with `ZKS_ENABLE_TEST_UI=ON`

## Release-Scope Hardening

This release also tightens a few release-readiness edges:

- empty `api_key` values are rejected instead of silently disabling auth
- non-loopback binds without API key auth emit a startup warning
- the public docs now describe the actual supported API subset
- license and release-planning documentation are now present in the repo

## Known Limitations

- sessions are in-memory and process-lifetime only
- the server runs a single shared `zoo::Agent`
- `/v1/tools` is empty in the default build because no server-owned tools are
  registered yet
- `/metrics` counts requests and errors from HTTP response status and does not
  fully describe failures that happen after an SSE stream starts
- on macOS with Metal enabled, certain OOM conditions can abort the process;
  see [zoo-keeper-metal-oom-issue.md](zoo-keeper-metal-oom-issue.md)
